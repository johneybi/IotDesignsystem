# IoT Design System Component Structure

이 문서는 IoT 디자인 시스템의 컴포넌트 폴더 구조와 각 디렉토리의 역할, 그리고 설계 의도를 설명합니다.
IoT 애플리케이션은 물리적인 장치 제어(Control)와 상태 모니터링(Monitoring)이라는 특수한 목적을 가지므로, 이에 최적화된 구조로 설계되었습니다.

## 📂 Directory Structure

### 1. `foundations/` (기반 요소)

**역할**: 디자인 시스템의 가장 기초가 되는 "원자(Atom)" 단위의 스타일과 규칙을 정의합니다.
**필요성**:

- **일관성(Consistency)**: 모든 컴포넌트가 동일한 색상, 타이포그래피, 아이콘 스타일을 공유하여 브랜드 아이덴티티를 유지해야 합니다.
- **확장성**: 추후 테마(Dark/Light 모드) 변경이나 브랜드 컬러 변경 시, 이곳의 토큰만 수정하면 전체 시스템에 반영됩니다.
- **포함 요소**: `Colors`, `Typography`, `Shadows`, `Icons`, `Radius`

### 2. `inputs/` (입력 및 제어)

**역할**: 사용자가 시스템이나 장치에 명령을 내리는 모든 인터페이스 요소를 담당합니다.
**필요성**:

- **IoT 특화 제어**: 일반 웹과 달리 IoT는 "조명 켜기(Toggle)", "온도 조절(Slider/Knob)", "모드 변경(Segmented Control)" 같은 물리적 제어 메타포가 중요합니다.
- **조작감**: 사용자가 장치를 실제로 제어하고 있다는 확실한 피드백을 주는 UI가 필요하므로 별도로 관리합니다.
- **포함 요소**: `Switch`, `Slider`, `Knob`, `Button`, `TextField`, `Select`

### 3. `data-display/` (데이터 표시)

**역할**: 센서 데이터, 장치 상태, 로그 등 정보를 시각적으로 보여주는 요소들입니다.
**필요성**:

- **가독성**: 온도, 습도, 전력량 등 복잡한 수치 데이터를 사용자가 한눈에 파악할 수 있도록 직관적인 디자인이 필요합니다.
- **실시간성**: 실시간으로 변하는 데이터를 효율적으로 보여주기 위한 최적화된 컴포넌트가 필요합니다.
- **포함 요소**: `SensorCard`, `StatusBadge`, `Chart`, `List`, `Table`, `Avatar`

### 4. `feedback/` (피드백 및 상태 알림)

**역할**: 사용자의 동작에 대한 시스템의 반응이나 상태 변화(성공, 실패, 로딩, 경고)를 알립니다.
**필요성**:

- **비동기 통신 대응**: IoT 제어는 네트워크 지연이나 장치 응답 대기 시간이 발생할 수 있습니다. "명령 전송 중", "제어 실패" 등의 상태를 명확히 알려주어 사용자 불안을 해소해야 합니다.
- **안전 알림**: 화재 경보, 배터리 부족 등 중요한 경고를 즉각적으로 전달해야 합니다.
- **포함 요소**: `Spinner(Loading)`, `Toast`, `Alert`, `Skeleton`, `ProgressBar`

### 5. `layout/` (레이아웃)

**역할**: 화면의 전체적인 구조를 잡고 컴포넌트들을 배치하는 컨테이너 역할을 합니다.
**필요성**:

- **반응형 대시보드**: 모바일, 태블릿, 월패드 등 다양한 디바이스 크기에 맞춰 그리드와 배치가 유연하게 동작해야 합니다.
- **공간 그룹화**: "거실", "침실" 등 공간 단위로 장치를 묶어서 보여주는 카드형 레이아웃 패턴이 자주 사용됩니다.
- **포함 요소**: `Grid`, `Stack`, `Container`, `Box`, `Divider`

### 6. `navigation/` (네비게이션)

**역할**: 앱 내의 페이지 이동이나 계층 구조 탐색을 돕습니다.
**필요성**:

- **다계층 구조**: "집 > 방 > 장치 > 상세 설정"으로 이어지는 깊은 뎁스를 쉽게 오갈 수 있어야 합니다.
- **포함 요소**: `Sidebar`, `Tabs`, `Breadcrumbs`, `BottomNavigation`

### 7. `overlay/` (오버레이)

**역할**: 현재 화면 위에 띄워지는 보조 화면들입니다.
**필요성**:

- **컨텍스트 유지**: 장치 상세 설정을 위해 페이지를 이동하기보다, 현재 대시보드 위에서 팝업(Drawer/Modal)으로 빠르게 설정을 바꾸고 닫는 경험이 IoT UX에서 효율적입니다.
- **포함 요소**: `Modal`, `Drawer`, `Tooltip`, `Popover`

---

## 🛠 `styles/`, `hooks/`, `utils/`

- **`styles/`**: 전역적으로 사용되는 CSS 파일, Reset CSS, Theme Provider 등을 관리합니다.
- **`hooks/`**: `useDeviceStatus`, `useMqtt` 등 IoT 데이터 통신과 관련된 재사용 가능한 로직을 모아둡니다.
- **`utils/`**: 데이터 포맷팅(예: 온도 단위 변환 `C` -> `F`), 날짜 변환 등 순수 함수들을 관리합니다.
